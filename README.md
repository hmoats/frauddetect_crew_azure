# FraudDetect Flow

A multi-crew fraud detection pipeline built with [CrewAI Flows](https://docs.crewai.com/concepts/flows). This project orchestrates 4 specialized AI crews through a structured, event-driven workflow with Python data validation between each step.

## Architecture

The flow processes transaction data through the following pipeline:

1. **Data Analysis Crew** (gpt-4o) -- Fetches dataset, trains RandomForest model via AutoML, produces predictions
2. **Model Validation** (Python) -- Checks for data leakage, validates metrics, routes based on quality
3. **Alert Triage Crew** (gpt-4o-mini) -- Triages real model predictions into autonomous/analyst/monitoring paths
4. **Investigation Crew** (gpt-4o) -- Provides evidence-based investigation support for complex alerts
5. **Performance Analytics** (Python) -- Deterministic cost/ROI calculations using real alert counts
6. **Compliance Crew** (gpt-4o + gpt-4o-mini) -- Algorithm explainability and ISO 42001 compliance assessment
7. **Final Report** (Python) -- Assembles all outputs into a comprehensive markdown report

## Key Improvements Over V1

- **Structured state** -- Pydantic models carry real data between steps (no LLM hallucination)
- **Data validation** -- Python checkpoints between crews catch data leakage, missing data, metric anomalies
- **Conditional routing** -- Router stops pipeline on poor model quality instead of propagating errors
- **Real predictions** -- Downstream crews work with actual model output, not invented customer data
- **Cost-optimized LLMs** -- gpt-4o for reasoning-heavy tasks, gpt-4o-mini for structured/lighter tasks
- **Deterministic analytics** -- Performance metrics calculated in Python, not generated by LLM

## Installation

Requires Python >=3.10, <3.14 and [uv](https://docs.astral.sh/uv/).

```bash
# Install uv if not already installed
pip install uv

# Install dependencies
crewai install
```

## Configuration

Copy `.env.example` to `.env` and add your API key:

```bash
cp .env.example .env
# Edit .env and set OPENAI_API_KEY
```

## Running

```bash
# Run the full pipeline
crewai run

# Visualize the flow graph
crewai flow plot
```

## Running Tests

```bash
uv run pytest tests/ -v
```

## Customizing

- **Dataset**: Change inputs in `main.py` `kickoff()` function
- **LLM models**: Adjust per-crew in each crew's `.py` file
- **Alert thresholds**: Modify triage rules in `alert_triage_crew/config/tasks.yaml`
- **Cost parameters**: Adjust constants in `main.py` (`ANNUAL_ALERTS_PROJECTION`, etc.)
